## 8. Design Theme

A direction that reads **premium, modern, and developer-credible** — dark-first with a confident accent. It signals "senior, design-minded engineer" without the generic purple-gradient AI look.

### Concept: *"Midnight Studio"*
Dark canvas, warm-to-electric accent, lots of breathing room, crisp typography, and restrained motion. Think a developer's portfolio that feels like a product, not a template.

### Color Palette

| Role | Hex | Use |
|---|---|---|
| Background (base) | `#0B0D10` | Page background, near-black with a cool tint |
| Surface | `#14171C` | Cards, panels |
| Surface elevated | `#1C2027` | Hover states, modals |
| Border / hairline | `#262B33` | 1px dividers, card outlines |
| Text primary | `#F4F6F8` | Headlines, body |
| Text muted | `#8B93A1` | Secondary text, captions |
| **Accent (primary)** | `#6366F1` | Indigo — links, buttons, highlights (matches your GitHub README) |
| Accent (secondary) | `#22D3EE` | Cyan — gradients, hover glow, code accents |
| Success / Vue nod | `#42B883` | Subtle Vue-green for tags/details |

**Gradient for hero text / buttons:** `linear-gradient(120deg, #6366F1 0%, #22D3EE 100%)`

> Optional **light mode**: invert to `#FAFAFA` background, `#0B0D10` text, keep the same indigo accent. Offer a theme toggle — on-brand for a Vue dev. **(Decided: dark-first with a light/dark toggle.)**

### The look we're going for: *"Framer-built minimalism"*
The target aesthetic is an **Awwwards / Linear / Framer-template** feel — premium, restrained, and obsessively smooth. Lots of negative space, oversized type, hairline glass surfaces, and motion that feels *physical* rather than flashy. **Minimal, but alive.** Nothing should ever just "appear" or snap — everything eases in with weight. If it looks like a Bootstrap template with nice fonts, it's wrong.

**Three rules that sell the "expensive" feel:**
1. **Space is the feature** — double your instinct for padding (`py-32`+ sections). Let it breathe.
2. **Motion has mass** — no `linear`/`ease-in-out`. Use spring-like custom curves so things settle, not stop.
3. **Surfaces are layered, not flat** — cards sit *inside* enclosures (see Double-Bezel below), with inner highlights instead of harsh shadows.

### Typography
- **Display / Headings:** `Clash Display` or `General Sans` (wide, geometric, confident) — via Fontshare.
- **Body:** `Satoshi` (clean, modern, premium) — via Fontshare. *(Avoid Inter/Roboto/Helvetica — they read "default/template".)*
- **Code / eyebrows / accents:** `JetBrains Mono` or `Fira Code` — keep your GitHub through-line.
- **Scale:** huge, tight headlines (`clamp(2.5rem, 6vw, 5.5rem)`, tracking `-0.02em`), airy body (`1.6` line-height), tiny uppercase eyebrows (`text-[10px] tracking-[0.2em]`).

### Layout archetype
- **Hero — Editorial Split:** massive headline on the left half, a subtle glass card / portrait / availability badge on the right. Mesh-gradient glow orbs (indigo→cyan, very low opacity) drifting slowly in the background.
- **Projects — Asymmetrical Bento:** a grid of varied tile sizes (one `col-span-8 row-span-2` hero tile beside stacked `col-span-4` cards) — never a boring even 3-column row. Collapses to single column below `768px`.
- Max content width ~1200px, section padding `py-28`–`py-40`.
- Rounded corners: exaggerated squircles (`rounded-[1.5rem]`–`rounded-[2rem]`).

### Component patterns (what makes it feel hand-built)
- **Double-Bezel cards:** every major card = an outer shell (`bg-white/5`, `ring-1 ring-white/10`, `p-2`, `rounded-[2rem]`) wrapping an inner core with its own background, an inner highlight (`shadow-[inset_0_1px_1px_rgba(255,255,255,0.12)]`), and a slightly smaller radius. Looks like glass set in machined metal.
- **Island buttons:** fully-rounded pills (`rounded-full px-6 py-3`). A trailing arrow (`↗`) lives in its *own* little circle nested flush-right, never naked next to the text.
- **Eyebrow tags:** pill-shaped micro-labels above each H2 (`SELECTED WORK`, `ABOUT`, etc.).
- **Hairlines + inner glows** instead of `shadow-md` / dark drop shadows.
- **Icons:** ultra-light line icons only (Phosphor Light / Remix Line) — never thick Lucide/FontAwesome.

### Motion choreography (minimalist, but premium)
Use a smooth-scroll layer + scroll-triggered reveals. Keep each effect subtle; the *consistency* is what reads as expensive.
- **Smooth scroll:** `Lenis` for that buttery Framer-style inertia scroll.
- **Custom easing everywhere:** `cubic-bezier(0.32, 0.72, 0, 1)` (the "Vaul/Linear" curve) over `700–900ms`.
- **Masked headline reveal:** hero lines slide up from behind an invisible mask (`translate-y-[110%] → 0`), staggered ~80ms per line, on load.
- **Scroll entry:** sections fade-up + de-blur (`translate-y-16 blur-md opacity-0 → translate-y-0 blur-0 opacity-100`) as they enter view. Use Framer Motion `whileInView` / `IntersectionObserver` — never a raw scroll listener.
- **Magnetic buttons & cards:** subtle pull toward the cursor on hover; press-down `active:scale-[0.98]`; the nested arrow circle drifts `translate-x-1 -translate-y-px` on hover.
- **Project cards:** gentle lift + image scale (`scale-105`) on hover, custom-eased.
- **Floating glass nav:** detached pill (`mt-6 mx-auto w-max rounded-full backdrop-blur`), hides on scroll-down / reveals on scroll-up. Mobile menu morphs the hamburger into an `X` and opens a full-screen blurred overlay with staggered link reveals.
- Optional: a slow tech-stack marquee strip, and a typing-effect tagline (you already do this on GitHub).
- **Always** respect `prefers-reduced-motion` (disable transforms/blur, show content immediately).

### Performance guardrails (so the animations stay smooth on mobile)
- Animate **only** `transform` and `opacity` — never `width`/`height`/`top`/`left`.
- `backdrop-blur` only on fixed/sticky elements (nav, overlays) — never on scrolling content.
- Grain/noise overlay only as a `fixed`, `pointer-events-none` layer.
- `will-change: transform` sparingly, only on actively-animating elements.

### Build stack — **Next.js (required)**
This portfolio **must be built with Next.js**. It matches the Framer aesthetic, deploys cleanly on Vercel, and — since your public work leads Vue-heavy — it deliberately showcases your React/Next.js side.

**The stack:**
- **Next.js (App Router)** — the framework. *Non-negotiable for this build.*
- `TypeScript`
- `TailwindCSS` — styling
- `Framer Motion` — masked reveals, `whileInView` scroll entries, magnetic/spring physics
- `Lenis` — buttery smooth-scroll
- `next/image` — optimized project shots & portraits
- `MDX` or `Contentlayer` — optional blog
- Ultra-light line icons (`@phosphor-icons/react` light weight)

→ **Deploy on Vercel** (first-party Next.js hosting; instant previews on every push).

> Scaffold with `npx create-next-app@latest --typescript --tailwind --app`.

**Reference vibe:** Linear.app, Framer.com templates, Vercel.com, Family.co — that tier of restraint + motion.

## 9. AI Photoshoot Prompts (likeness-locked to your reference photo)

**Workflow: you upload a reference image of yourself, and the model recreates *you* in new scenes — same skin tone, same facial structure, same features.** These are image-to-image / character-reference prompts, not text-only generation.

**Best tools for high-fidelity likeness (pick one):**
- **Google Gemini ("Nano Banana" / Imagen image editing)** — upload your photo and prompt the new scene; excellent at holding identity. *(You already build with Gemini — natural fit.)*
- **Midjourney** with `--cref [your_photo_url]` (character reference) + `--cw 100` (max character weight = preserve face/features most strongly).
- **Flux (Kontext / PuLID)** or a **LoRA trained on 10–15 photos of you** — the gold standard for exact likeness if you want many shots.

> **How to use:** upload **1 clear, front-facing, well-lit photo** (more is better — 3–5 angles for LoRA). Paste the **Identity Lock** block + a scene prompt below. Keep the bracketed wardrobe/scene bits; the identity line does the heavy lifting.

**🔒 Identity Lock (paste at the START of every prompt):**
```
Use the uploaded photo as the exact reference for this person. Preserve their identity
precisely: same dark/deep brown skin tone (do NOT lighten, whiten, or change skin color),
same facial structure and bone structure, same nose, lips, eyes, eyebrows, jawline,
hairline and hair texture, same complexion and natural skin texture. This is the SAME
person — only change the scene, lighting, wardrobe and pose described below. Keep an
accurate, true-to-life likeness; do not beautify, slim, or alter the face.
```

> ⚠️ The single most important instruction is **"do not change skin color / do not lighten."** Many models drift lighter by default — keep that line in every prompt, and reject any output where your complexion looks off.

**Settings tips**
- **Likeness first:** keep the 🔒 Identity Lock block in *every* prompt. After each generation, sanity-check skin tone, nose, lips and jawline against your reference — discard any that drifted lighter or "prettied up" your features.
- Use a **clear, high-res, front-facing reference** with even lighting (no heavy filters/sunglasses). For LoRA/PuLID, give 5–15 varied angles for the most consistent results.
- Keep lighting/background consistent across all four shots so the site feels cohesive (the indigo→cyan accent ties them to the design theme).
- Generate 4–8 variants per prompt, pick the most natural — avoid the "too-perfect plastic skin" ones.
- Negative prompt (where supported): `lightened skin, whitewashed, changed skin tone, different face, plastic skin, over-smoothed, beautified, cartoon, distorted hands, extra fingers, watermark, text, logo, asymmetric eyes`.
- **Midjourney:** append `--cref [your_photo_url] --cw 100 --style raw --ar 4:5` (`--cw 100` = preserve face/features most strongly).
- **Gemini / Nano Banana:** upload your photo, then prompt *"keep this exact person and their skin tone, change only the scene to…"* — re-roll if identity drifts.
